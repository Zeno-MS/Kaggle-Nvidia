## Pattern Analysis

### What approaches are competitors likely using?
- Standard CoT prompting (most common, lowest effort)
- Synthetic data + fine-tuning (medium effort, AIMO-2 inspired)
- Self-consistency / majority voting (common, moderate gains)

### What approaches are competitors unlikely using?
- Symbolic program synthesis (requires different skillset)
- Judge-specific optimization (requires understanding reward model internals)
- Cognitive trace templates (requires cross-disciplinary thinking)

### Our differentiation moat
The three unconventional edges are unlikely to be independently discovered by
many competitors within the competition timeframe, because:
1. Judge reverse-engineering requires knowing about HelpSteer weights (obscure)
2. Symbolic solving requires program synthesis expertise (rare in Kaggle ML community)
3. Cognitive trace design requires connecting cognitive science to LLM evaluation (novel)
